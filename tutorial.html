📖 使い方ガイド

LP StructureMaker ― はじめてでも、選んで作れる。LPの「構成の設計図」から画像まで揃います。 無料 / 60秒

✨これは何のツール？

「LPって、何をどの順番で書けばいいの？」に答えるツールです。 業種・読み手・ゴールの3つを選ぶだけで、あなたのLPに合った 構成（骨組み）の設計図が完成。右側の見取り図で、LPの形が目でわかります。

このツール自体はAIを使いません（＝設計図の自動販売機です）。だから何回使っても無料。 できあがった文章を ChatGPT や Claude に貼ると、原稿やLPそのものに変わります。

🗂️はじめに：フォルダとリポジトリを用意する（初回だけ）

プロンプトをAIに貼る前に、保存先を先に作っておくと、あとで画像zipを展開して置くだけ・GitHubにpushするだけで 公開まで進められます。すでに作業用フォルダがある人はこのカードは読み飛ばしてOK。

  1. 1

    フォルダを作る

    ターミナル、または Claude Code / Codex のようなAIエージェントに、下のプロンプトをそのまま貼るだけでOK。

  2. 2

    GitHubにリポジトリを作る

    github.com/new から手動で作るか、 エージェントに任せるなら続けて下のプロンプトを貼る。

  3. 3

    作業に入る

    ここまでできたら、下の「最短の道」からツールを使い始めてOK。画像ができたら書き出したZIPをこのフォルダに展開してpushするだけ。

🤖 AIエージェント（Claude Code / Codex など）に貼るプロンプト

新しいプロジェクト用フォルダ「(サイト名をここに)」を作成し、その中に移動して git init を実行してください。
続けて GitHub に同名の空リポジトリを作成してください
（gh コマンドが使えるなら `gh repo create (サイト名) --public --source=. --remote=origin --push` を実行、
使えない場合は https://github.com/new から手動で作成してもらい、できたリポジトリのURLを私に教えてください）。
リポジトリができたら、このフォルダをそのリモート（origin）にひも付けて、空コミットを1つ push してください。
終わったら「フォルダとリポジトリの準備ができました」と教えてください。

⌨️ 手動でやる場合（ターミナル）

mkdir (サイト名) && cd (サイト名)
git init
# → https://github.com/new で同名のリポジトリを作成したあと
git remote add origin https://github.com/(あなたのアカウント名)/(サイト名).git
git add . && git commit -m "first commit"
git push -u origin main

💡 このフォルダに、②で書き出す画像の zip を展開し、AIに作らせた index.html を置いて push すれば、 そのまま GitHub Pages などで公開できる状態になります。

⚡最短の道：3ステップ（約60秒）

迷ったらこれだけ。プリセットを押すだけで完成＆コピーまで終わります。

  1. 1

    プリセットを選ぶ

    左上から、自分の商売に近いものを1つクリック。 業種・読み手・ゴール・トーンが一瞬で埋まり、設計図が自動でコピーされます。

  2. 2

    見取り図で確認

    右側にLPの形が積み木で表示されます。ブロックをクリックすると、 設計図の「そのセクションに何を書くか」がハイライトされます。

  3. 3

    AIに貼る

    ChatGPT か Claude を開いて貼り付け、送信。 「この構成でLPの原稿を書いて」と一言添えれば、原稿まで書いてくれます。

💡 迷ったら「おまかせ」ボタン。相性の良い組み合わせが1つ出ます。 業種・読み手・ゴールを選び直すと、見取り図と設計図がその場で組み変わります。

📤出力タイプ：2つの使い方

基本設定のいちばん下「出力タイプ」で切り替えます。

🧱 構成の設計図（まず考えたい人）

LPに載せるセクションの順番と、各セクションに「何を書くか」のガイド。 自分で文章を書く指針にもなるし、AIに渡して原稿を書いてもらう土台にもなります。

🚀 LP制作プロンプト（そのまま作りたい人）

AIにLPそのもの（動くWebページ）を作らせる指示文です。 Claude / ChatGPT / v0 に貼ると、その場でLPのプレビューが表示されるよう指示済み。 コードがわからなくても、貼るだけで「動くLP」が目の前に出てきます。

🖼️ 必要な画像プロンプト集（画像を用意する人）

LPができたら、次は画像。この出力に切り替えると、このLPに必要な画像の一覧（メインビジュアル・メニュー写真・雰囲気カットなど）と、 1枚ずつコピーして使える生成プロンプトが出ます。業種に合ったテイストが固定済みなので、 全部同じスタイル指定で作れば世界観が揃います。

おすすめの流れ：まず「構成の設計図」で骨組みに納得 → 「LP制作プロンプト」に切り替えてコピー → AIへ。 切り替えても選択内容はそのまま引き継がれます。

🔧こだわる道：作り込みたい人へ

最短の道はそのままに、必要な人だけ下の要素を足していけます。

🏷️ 掲載情報を入れる（店名・価格・日時など）

折りたたみを開いて店名・価格・日時・場所・特徴を入れると、設計図と制作プロンプトに 「この情報を正確に載せる」指定が入ります。 空欄のままなら「【店名】のように仮置きして、架空の実績を作らない」という安全指示が自動で入るので、 AIが勝手に嘘の情報を作る事故を防げます。

👤 読み手で構成が変わる

「まだ知らない人」なら共感とベネフィットが厚く、「迷っている人」なら実績とFAQが厚く、 「あと一押しの人」なら料金と流れが前に、「常連さん」ならお知らせ・特典が最初に来ます。 読み手を選び直して、見取り図の変化を見てみてください。

🖼️ 画像は姉妹ツールで（Visual PromptMaker 連携）

設計図には、業種に合った写真のテイスト（例：飲食なら“シズル感のある料理写真”）が書かれています。 そのテイストのまま Visual PromptMaker の Graphic タブで画像プロンプトを作れば、 LP全体の世界観が揃います。

🏪業種で構成はこう変わる（早見表）

「悩み解決」の型を全業種に当てはめない、が このツールの考え方です。

業種構成の主役
飲食店・カフェ・バーコンセプト → 看板メニュー → 雰囲気
サロン・整体・クリニック共感（悩み）→ 選ばれる理由 → 実績
SaaS・アプリ課題 → 機能 → 料金プラン
EC・物販ベネフィット → こだわり → レビュー
セミナー・講座得られる未来 → カリキュラム → 講師
コーポレート・個人提案の核 → 強み → 実績

🖼️プリセットの見本画像を足す（管理者向け）

見本画像が無いプリセットは絵文字で表示されます。画像を足すには：

  1. Visual PromptMaker などでLPの完成イメージ画像を作る。
  2. 長辺720px・jpg（150KB以下が目安）に整え、 assets/presets/lp/<プリセットID>.jpg（例 resto.jpg）で保存。
  3. ツールを開き直すと、サムネイル・拡大・仕上がり例に自動で反映されます（コード変更は不要）。
