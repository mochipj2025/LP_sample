# v2 ― 情報設計やり直し用の作業場所

既存の `index.html`（トップ階層）はそのまま動く状態で残す。ここは画面構成そのものを
作り直すための、別の入口。ビジネスロジック（`js/core.js` / `js/template.*.js`）は
複製せず、相対パスでそのまま読み込む。重複ファイルを増やさないため。

## 今回のきっかけ

「プリセットがわかりにくいかも」という指摘から、画面全体の情報設計を見直すことになった。

現状の問題（ユーザー確認済み）:
- 情報過多・ごちゃごちゃ（1画面に20項目以上）
- タブ切替やナビが分かりにくい（タブ3つ＋ヘッダーリンク3つが横並び）
- 入力項目が多すぎる・迷う（基本設定だけでチップ6グループ）
- プリセットは「カード本体クリック＝即・適用＋自動コピー」だが、中身を事前確認する
  手段（虫めがねボタン）は画像が読み込めたときしか出ない。見た目は写真ギャラリー、
  実態は問答無用の即時上書きというギャップがある

## 実装済みの流れ

下記はすべて実装・動作確認済み（`v2/index.html` の各 `<section>` / `v2/js/app.js` 対応）。

```
スタート
  └─ ⓪ folderStep：フォルダを準備する（初回だけ／スキップ可）
       今まで tutorial.html にしか無かった「はじめに：フォルダとリポジトリを
       用意する」を、ツール自体のステップ0として組み込んだ。中身は
       <details> に畳んであり、読まずにコピペするだけで進められる。
  └─ entryStep：プリセットを使う？
       ├─ はい → presetStep（アイコン＋名前＋説明文を常時表示。画像の読込
       │         成否で見た目やクリック挙動が変わらない）
       │         → クリックで confirmStep へ（ここで初めてコピー）
       └─ いいえ → basicStep（①業種・読み手・ゴールの3つだけ）
                 → imageStep（②画像・任意。template.imageSlots があるときだけ）
                 → infoStep（③掲載情報・任意。店名／商品・メニュー／日時／
                             場所／特徴・こだわり／リンク。ラベル文言は
                             template.inputLabelOf(key, state) で業種ごとに
                             差し替わる＝メニュー欄が「看板メニュー」「料金
                             プラン」等に、特徴欄も業種の言葉になる）
                 → advancedStep（④詳細設定・任意。トーン・ボリューム・
                                 出力タイプ・参考・補足）
                 → confirmStep（内容確認・コピー）
```

`confirmStep` の「← 選び直す」は `confirmReturnTo` に入っている直前のステップへ戻る
（プリセット経由なら presetStep、手動ルートなら advancedStep）。

## まだ決めていないこと

- chips 系フィールド（トーン等）の自由入力（`allowCustom`）は v2 では未実装。
  v1 にはあるが、basicStep・advancedStep ともに「あらかじめ用意した選択肢から選ぶ」
  だけにしてある（素人が一発でわかる、を優先した意図的な簡略化。困る人が出てきたら
  v1 と同じ自由入力欄を足す）
- タブ（LP作成／画像作成／アイコン作成）＋ヘッダーリンク（使い方／サンプル／
  ステップ版）を v2 にどう持ち込むか。今は LP作成1本のみ対応
- v2が固まったあと、トップ階層（`index.html` / `wizard/`）とどう入れ替える／
  共存させるか
